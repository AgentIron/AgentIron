## ADDED Requirements

### Requirement: AgentIron SHALL support plugin auth flows
AgentIron SHALL expose client-driven commands for starting and completing plugin authentication using the `iron-core` auth interaction contract.

#### Scenario: User starts plugin authentication
- **WHEN** the frontend requests authentication for a plugin that requires auth
- **THEN** the backend SHALL return the auth interaction request produced by `iron-core`

#### Scenario: User completes plugin authentication
- **WHEN** the frontend submits an auth interaction response for an authenticating plugin
- **THEN** the backend SHALL complete the flow through `iron-core` and return the resulting auth status transition

### Requirement: AgentIron SHALL surface plugin auth state to the UI
AgentIron SHALL expose auth prompts and auth state transitions so the frontend can identify which plugins are blocked on authentication.

#### Scenario: Auth prompts are requested
- **WHEN** the frontend requests plugin auth prompts for an active session
- **THEN** AgentIron SHALL return the current prompts generated by `iron-core`

#### Scenario: Auth state changes during a session
- **WHEN** `iron-core` emits an auth state transition for a plugin
- **THEN** AgentIron SHALL forward that transition to the frontend with the plugin identifier and both states

### Requirement: AgentIron SHALL preserve rich plugin tool results
AgentIron SHALL forward transcript-safe text and structured rich view payloads from plugin tool results without discarding them.

#### Scenario: Plugin tool returns transcript text and view data
- **WHEN** `iron-core` emits a tool result containing `transcript_text` and `view`
- **THEN** AgentIron SHALL include both fields in the frontend event payload for that tool result

#### Scenario: Plugin tool returns plain JSON only
- **WHEN** a tool result has no rich view payload
- **THEN** AgentIron SHALL continue to render the standard result representation

### Requirement: AgentIron SHALL render supported plugin rich views
AgentIron SHALL render current `iron-core` rich result payload kinds in chat with a readable fallback for unknown payloads.

#### Scenario: Status view is returned
- **WHEN** a plugin tool result contains a status view payload
- **THEN** the chat UI SHALL render the status text and level in a structured format

#### Scenario: Progress view is returned
- **WHEN** a plugin tool result contains a progress view payload
- **THEN** the chat UI SHALL render a progress indicator using the reported value

#### Scenario: Todo list view is returned
- **WHEN** a plugin tool result contains a todo list view payload
- **THEN** the chat UI SHALL render the list items with their completion state
